## 1. 产品概述

### 1.1 产品定位

**AI Story Studio** 是一款面向中文网文作者与内容创作者的 AI 辅助写作工具。产品核心理念是"**AI 辅助，作者掌控**"——通过智能大纲、章节开头生成、多版本续写建议等功能，帮助作者提升创作效率，同时保持创作主导权。

### 1.2 核心价值主张

| 价值点 | 描述 |
|--------|------|
| **大纲驱动** | 三幕式故事结构 + 分章大纲，规划清晰 |
| **开头先行** | 每章只生成 100 字开头，作者继续创作 |
| **多版本选择** | 同一动作 1/3/5 个候选，降低跑偏率 |
| **选区改写** | 精确控制 AI 作用范围（选区/光标/全章） |
| **一致性保障** | 作品档案自动注入，长篇不遗忘 |
| **多端同步** | 本地优先 + 可选 OSS 云端备份 |

### 1.3 与竞品差异

| 维度 | Sudowrite | AI Story Studio |
|------|-----------|-----------------|
| 目标市场 | 英文小说作者 | 中文网文作者 |
| 生成策略 | 可生成全章 | 只生成开头，作者续写 |
| 大纲功能 | Scene Card | 三幕式大纲 + 章节大纲 |
| 一致性 | Canvas/Story Bible | 作品档案 + 自动注入 |
| 部署方式 | SaaS | 本地优先 + 可选云端 |

## 2. 目标用户

### 2.1 用户画像

| 画像 | 描述 | 核心需求 |
|------|------|----------|
| **网文作者** | 日更 2000-8000 字，多部作品并行 | 快速出稿、情节连贯、批量操作 |
| **业余创作者** | 周更或随缘更，单部作品 | 降低创作门槛、灵感激发 |
| **内容团队** | 多人协作，质量把控 | 统一风格、版本管理、进度追踪 |

### 2.2 使用场景

1. **新书启动**：输入书名+简介 → 生成故事结构 → 生成分章大纲 → 批量生成开头
2. **日常创作**：打开章节 → 基于开头续写 → AI 辅助润色/扩写 → 完成发布
3. **瓶颈突破**：选中卡壳段落 → AI 续写/改写 → 多版本对比 → 采纳满意版本
4. **质量提升**：选中平淡描写 → AI 增强（环境/情绪/动作） → 替换原文

### 3.3 AI 写作助手

#### 3.3.1 AI Actions 动作体系

| 分类 | 动作 | 输入范围 | 输出长度 | 候选数 |
|------|------|----------|----------|--------|
| **推进** | 续写 | 末尾 800 字 | 100/300 字 | 1/3/5 |
| **推进** | 写开头 | 章节大纲 | 100 字 | 1 |
| **推进** | 生成标题 | 前 600 字 | 10 字内 | 1 |
| **改写** | 润色 | 选区/全章 | 原长度 | 1/3/5 |
| **改写** | 精简 | 选区/全章 | 缩减 20% | 1/3/5 |
| **改写** | 扩写 | 选区/全章 | 扩展 30% | 1/3/5 |
| **描写** | 环境 | 选区 | 100-180 字 | 1/3/5 |
| **描写** | 情绪 | 选区 | 100-180 字 | 1/3/5 |
| **描写** | 动作 | 选区 | 100-180 字 | 1/3/5 |
| **描写** | 对话 | 选区 | 100-180 字 | 1/3/5 |

#### 3.3.2 动作输入模型

```typescript
interface ActionRequest {
  novelId: string           // 小说 ID
  chapterId?: string        // 章节 ID
  action: ActionType        // 动作类型
  input: {
    scope: 'selection' | 'cursor' | 'chapter'  // 作用范围
    text: string            // 输入文本
    selection?: { start: number; end: number } // 选区位置
  }
  options: {
    variants: 1 | 3 | 5     // 候选数量
    length?: '100' | '300'  // 输出长度
    style?: string          // 风格提示
    subAction?: string      // 子动作（如 describe 的具体类型）
  }
}
```

#### 3.3.3 动作输出模型

```typescript
interface ActionResponse {
  success: boolean
  candidates: Array<{
    text: string            // 生成文本
  }>
  usage?: {
    promptTokens: number
    completionTokens: number
    totalTokens: number
  }
}
```

| 操作 | 描述 | 交互 |
|------|------|------|
| 替换选区 | 用候选替换选中内容 | 一键替换 |
| 插入光标 | 在光标处插入候选 | 一键插入 |
| 追加末尾 | 追加到章节末尾 | 一键追加 |
| 复制 | 复制到剪贴板 | 一键复制 |

### 3.4 大纲系统

#### 3.4.1 故事结构（三幕式）

```typescript
interface StoryStructure {
  beginning: string   // 开头阶段（15% 章节）- 人物登场、背景、初始冲突
  middle: string      // 经过阶段（70% 章节）- 发展、转折、成长
  ending: string      // 结尾阶段（15% 章节）- 高潮、结局
}
```

#### 3.4.2 章节大纲

```typescript
interface ChapterOutline {
  index: number           // 章节序号
  title: string           // 章节标题
  outline: string         // 情节概述（40-80 字）
  estimatedWords: number  // 预估字数
  phase: 'beginning' | 'middle' | 'ending'  // 所属阶段
}
```

#### 3.4.3 大纲生成流程

```
用户输入（标题+简介+体裁+字数+章节数）
           ↓
    ┌──────────────┐
    │  生成故事结构  │  ← AI 生成三幕概述
    └──────────────┘
           ↓
    ┌──────────────┐
    │  分批生成大纲  │  ← 每 20 章一批，带阶段上下文
    └──────────────┘
           ↓
    ┌──────────────┐
    │  大纲预览编辑  │  ← 用户可修改每章标题/大纲
    └──────────────┘
           ↓
    ┌──────────────┐
    │  批量生成开头  │  ← 每章约 100 字开头（跳过已有章节）
    └──────────────┘
```

#### 3.5.2 自动注入策略

| 场景 | 注入内容 |

## 5. API 规格

### 5.1 AI Action API

**端点**: `POST /api/ai/action`

**请求体**:
```json
{
  "novelId": "clx123...",
  "chapterId": "clx456...",
  "action": "continue",
  "input": {
    "scope": "chapter",
    "text": "已有内容..."
  },
  "options": {
    "variants": 3,
    "length": "300"
  }
}
```

**响应**:
```json
{
  "success": true,
  "candidates": [
    { "text": "续写内容 1..." },
    { "text": "续写内容 2..." },
    { "text": "续写内容 3..." }
  ]
}
```

### 5.2 大纲生成 API

**端点**: `POST /api/ai/generate-outline`

**请求体**:
```json
{
  "title": "剑破苍穹",
  "description": "少年从山村走出...",
  "genre": "玄幻",
  "totalWords": 100000,
  "chapterCount": 20
}
```

**响应**:
```json
{
  "success": true,
  "outline": {
    "beginning": "开头阶段概述...",
    "middle": "经过阶段概述...",
    "ending": "结尾阶段概述...",
    "chapters": [
      { "index": 0, "title": "第1章 山村少年", "outline": "...", "estimatedWords": 5000 }
    ],
    "totalWords": 100000,
    "chapterCount": 20
  }
}
```

### 5.3 流式生成 API

**端点**: `POST /api/ai/stream-generate`

**SSE 事件**:
| 事件 | 描述 | 数据 |
|------|------|------|
| `start` | 开始生成 | `{ message, totalChapters }` |
| `structure` | 故事结构完成 | `{ beginning, middle, ending }` |
| `batch_start` | 开始一批大纲 | `{ batch, totalBatches }` |
| `outlines` | 大纲生成完成 | `{ chapters[] }` |
| `chapter_start` | 开始生成章节 | `{ index, title }` |
| `chapter_done` | 章节完成 | `{ index, title, wordCount, progress }` |
| `complete` | 全部完成 | `{ totalChapters, totalWords }` |
| `error` | 出错 | `{ error }` |

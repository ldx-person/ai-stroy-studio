# AI Story Studio · 产品功能描述

## 产品定位

**AI Story Studio** 是一款面向中文网文作者与内容创作者的 AI 辅助写作工具。产品核心理念是"**AI 辅助，作者掌控**"——通过智能大纲、章节开头生成、多版本续写建议、描写增强等功能，帮助作者提升创作效率，同时保持创作主导权。

---

## 核心功能

### 一、小说与章节管理

- **多本小说管理**：创建、编辑、删除小说，支持标题、简介、类型（体裁）、状态（草稿 / 连载中 / 已完结）。
- **章节目录**：左侧目录展示所有章节及字数，支持新建、删除、切换章节；目录区域独立滚动，不拉长整页。
- **章节编辑**：每章独立编辑区，支持标题修改、正文编辑；内容自动保存，字数实时统计。
- **章节修订历史**：自动保存每次编辑的版本记录，支持查看历史版本并一键恢复。
- **数据持久化**：本地数据库（Prisma + SQLite）存储小说与章节；可选阿里云 OSS 同步，实现多端备份与恢复。

### 二、智能大纲与章节生成

- **检索现有目录**：生成前会读取当前小说的已有章节列表，按顺序从最后一章之后续写，**不覆盖已有章节**。
- **故事结构**：根据标题、简介、体裁、计划总字数与章节数，自动生成「开头 / 经过 / 结尾」三阶段故事概述。
- **章节大纲**：按阶段分批生成每章标题与情节大纲（约 40–80 字概述），保证前后连贯。
- **每章仅生成开头**：每章只生成约 **100 字**的章节开头（开篇段落），用于定调与引导，**不生成整章全文**，方便作者在此基础上续写或使用其他 AI 能力扩写。
- **批量生成**：支持一键批量生成所有章节的大纲和开头，跳过已有内容的章节。

### 三、AI 写作助手（Actions 体系）

AI 能力已统一整合为 **Actions 动作体系**，支持精确的输入范围控制和多版本候选：

#### 3.1 统一的动作输入模型

每个 AI 动作都支持：
- **输入范围**：`selection`（选区）、`cursor`（光标位置）、`chapter`（整章）
- **输出长度**：100 字 / 300 字
- **候选数量**：1 / 3 / 5 个版本（Variants）

#### 3.2 推进类动作

| 动作 | 描述 | 输入 | 输出 |
|------|------|------|------|
| **AI 续写** | 根据已有内容自然延续 | 末尾 800 字 | 100–350 字/版本 |
| **写开头** | 根据大纲写章节开头 | 章节大纲 | 约 100 字 |
| **生成标题** | 根据内容生成章节标题 | 前 600 字 | 10 字内（保留原章节号） |

#### 3.3 改写类动作

| 动作 | 描述 | 特点 |
|------|------|------|
| **润色优化** | 提升文笔表达，保持原意 | 语句更通顺、更有画面感 |
| **精简文本** | 保留关键信息，压缩篇幅 | 删除冗余，保持连贯 |
| **细节扩写** | 丰富场景、情绪与对话细节 | 不改变核心剧情 |

#### 3.4 描写增强（Describe）⭐ 新增

支持四种描写维度的增强，每种都有专业提示词引导：

| 类型 | 增强方向 | 示例效果 |
|------|----------|----------|
| **🌿 环境** | 天气、光线、声音、气味、空间氛围 | "房间里很暗" → "烛火摇曳，在斑驳的墙纸上投下诡异的影子" |
| **💭 情绪** | 内心活动、微表情、肢体语言、心理张力 | "她很生气" → "她的指甲深深掐进掌心，嘴角却勾起一抹冷笑" |
| **⚡ 动作** | 动作分解、节奏变化、力量感、画面感 | "他跑了过来" → "他三步并作两步冲过来，衣角带起一阵风" |
| **💬 对话** | 语气变化、潜台词、对话节奏、符合人物性格 | "他说好的" → "他顿了顿，声音低下去：'……好。'那尾音像是从牙缝里挤出来的" |

#### 3.5 多版本候选与对比

- **Variants 机制**：同一动作可生成 1/3/5 个候选版本，降低"跑偏"概率
- **差异对比视图**：原文与候选的 Diff 对比，方便评估修改幅度
- **一键采用**：支持替换选区、插入光标处、追加末尾

#### 3.6 一致性保障（Story Bible）

- **作品档案**：存储角色卡、世界观、时间线、文风规则等设定
- **自动注入**：AI 生成时自动拼接相关设定，减少长篇创作中的设定漂移
- **API 支持**：`/api/story-bible` 提供 GET/PUT 接口管理档案

### 四、语音朗读（TTS）

- **章节朗读**：对当前章节正文进行语音合成与播放。
- **多音色与语速**：支持多种中文音色（童童、吹吹、小晨、豆豆、罗多）及语速调节（0.5x–2.0x）。
- **分段播放**：长文自动分块（每块约 500 字）请求与连续播放，支持上一段 / 下一段切换。

### 五、体验与布局

- **响应式布局**：桌面端为「章节目录 + 编辑区 + AI 助手面板」三栏；移动端为抽屉式章节目录与底部 Tab 切换（写作 / AI 助手 / 大纲 / 语音）。
- **选区高亮**：选中文字后，AI 动作默认作用于选区，操作更精准。
- **字数展示**：目录中每章显示字数；编辑区实时显示当前章节字数。
- **章节内容补全**：当目录显示某章有字数但正文为空时，切换章节会自动请求接口拉取或从 OSS 补齐内容。
- **大纲面板**：独立的大纲 Tab，支持预览故事结构、章节列表、批量生成开头、编辑单章大纲。

## 技术架构

- **前端**：Next.js 16（App Router）、React 18、TypeScript 5、Tailwind CSS 4、shadcn/ui
- **后端**：Next.js API Routes、Prisma ORM、本地 SQLite；可选阿里云 OSS 存储与同步
- **AI 模型**：统一调用层，默认 Z.AI GLM-5-Turbo，支持可配置切换
- **部署**：支持 Node/Bun 运行；可配合 Docker、Railway 等部署方案

### API 概览

| 端点 | 描述 |
|------|------|
| `/api/ai/action` | 统一 AI 动作入口（续写/改写/描写/开头/标题） |
| `/api/ai/generate-outline` | 生成故事大纲 |
| `/api/ai/stream-generate` | SSE 流式批量生成章节 |
| `/api/story-bible` | 作品档案 CRUD |
| `/api/chapters/revisions` | 章节修订历史 |
| `/api/tts` | 语音合成 |

## 适用场景

- **网文作者**：快速搭建多本书、分章创作，用 AI 生成大纲与每章开头，再自行或结合 AI 续写、润色、描写增强。
- **内容团队**：统一管理多部作品与章节，利用 AI 助手提升成稿效率与一致性，通过作品档案保障设定统一。
- **个人创作者**：在单设备或配合 OSS 做简单多端备份，完成从构思到成稿的闭环。
